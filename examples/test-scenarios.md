# 测试场景剧本

> 本文档提供 4 个可手动运行的测试场景，用于验证 `case-method` skill 是否正确触发、正确路由到对应知识库文件、输出符合预期格式。
>
> **如何使用**：在安装好 skill 的 Claude Code 会话中，逐个输入下方"输入"字段的内容，然后对照"预期行为"清单核对输出。

---

## 测试 1：子命令路由正确性（design）

**输入**：
```
/case-method:design 我想研究东亚三个国家（台湾、韩国、日本）的民主化路径差异
```

**预期行为**：
- [ ] Claude 进入"阶段 A：研究谜题诊断"，提问因变量和对照空间
- [ ] **不应**一次性读取所有 8 个 references 文件（只在需要时按需读取）
- [ ] 分阶段推进，每阶段只提出 1-2 个关键问题，等待用户回答
- [ ] 在阶段 C（案例选择）时提示可能的 MSSD 设计
- [ ] 最终输出包含 8 个字段的"研究设计摘要"

**常见失败模式**：
- ❌ 一次性读完所有 references 文件（违反按需加载原则）
- ❌ 跳过交互直接给方案（除非用户显式说"快速给我方案"）
- ❌ 未识别这是 MSSD 的典型应用场景

## 测试 2：check 的六维度完整性（DV 选择偏差识别）

**输入**：
```
/case-method:check
我的研究设计：比较 5 个民主化成功的案例（台湾、韩国、西班牙、波兰、南非），
通过过程追踪识别它们共同的因果因素，以解释"什么导致了民主化"。
```

**预期行为**：
- [ ] 识别这是典型的 **DV 选择偏差**（因变量无变异）
- [ ] 在六维度中的"案例选择"维度标 🔴 **需修改**
- [ ] 引用 Geddes (1990) 的经典警告和 KKV (1994) 的系统阐述
- [ ] 在"综合评估"中将此列为 #1 优先问题
- [ ] 给出可操作的修改建议：纳入负面案例 + 应用可能性原则（Mahoney & Goertz 2004）
- [ ] 可能提示检验 *充分条件* 需要 DV 变异

**常见失败模式**：
- ❌ 只标 🟡（需注意），未识别这是严重方法论错误
- ❌ 未引用 Geddes 或 KKV
- ❌ 建议"再加一个负面案例"但未说明可能性原则

## 测试 3：lookup 的中英混合匹配

**输入**：
```
/case-method:lookup conceptual stretching
```

**预期行为**：
- [ ] 读取 `references/07_术语表与学者争论.md` 和 `references/01_概念构建与测量.md`
- [ ] 按术语格式输出（5 个字段）：定义 / 核心要点 / 来源 / 相关概念 / 在研究设计中的含义
- [ ] 来源标注 Sartori (1970)
- [ ] "相关概念"中至少提及：抽象阶梯（Ladder of Abstraction）、旅行问题（Travelling Problem）
- [ ] 中文主体，关键术语附英文原文

**常见失败模式**：
- ❌ 只给 2-3 句英文定义，未按 skill 规定的格式输出
- ❌ 遗漏"在研究设计中的含义"字段
- ❌ 未读取 references/01 提供深度上下文

## 测试 4：无子命令的被动路由

**输入**（不带 `/case-method` 前缀）：
```
我的案例研究要怎么选负面案例？
```

**预期行为**：
- [ ] 按 SKILL.md 的"问题类型路由"表匹配"选案例 / 负面案例"→ 读取 `references/03_案例选择与选择偏差.md`
- [ ] 主动给出 Mahoney & Goertz (2004) 的**可能性原则**框架
- [ ] 解释**纳入规则**和**排除规则**
- [ ] 区分范围条件 vs. 可能性原则的不同职责
- [ ] 在回答末尾建议运行 `/case-method:design` 获得完整研究设计引导

**常见失败模式**：
- ❌ 未触发 skill，给出一般性 Chat 回答
- ❌ 遗漏可能性原则（只说"选一些没发生结果的案例"）
- ❌ 未引用具体作者年份

## 进阶测试（可选）

### 测试 5：write 的章节辩护质量

**输入**：
```
/case-method:write 过程追踪
我的研究：2 个案例（台湾 2000 年、韩国 1987 年）的民主化，
我想用过程追踪检验"经济危机 → 精英分裂 → 民主化"的因果机制。
```

**预期行为**：
- [ ] 生成 300-500 字的方法论辩护段落（非模板化，基于用户具体案例定制）
- [ ] 引用 George & Bennett (2005)，区分过程追踪的四种类型
- [ ] 承认局限（如"过程追踪不等于堆砌更多观察"）
- [ ] 预见 2-3 个审稿人质疑 + 预防性回应

### 测试 6：学者争论的平衡呈现

**输入**：
```
/case-method:lookup KKV 和 George & Bennett 的分歧
```

**预期行为**：
- [ ] 按争论格式输出：立场 A / 立场 B / 核心张力 / 对研究设计的含义
- [ ] 不偏向任何一方，分别呈现 KKV（统一推断逻辑）和 G&B（共享认识论但不同方法论）的立场
- [ ] 提及这是本 skill 识别的"六大争论"中的第一个

## 反馈

如果测试中发现 skill 行为与预期不符，请在 [GitHub Issues](https://github.com/ChauMingTsun-lab/case-method-skill/issues) 中说明：

1. 运行的测试编号（或输入的原始文本）
2. 实际输出（至少粘贴关键段落）
3. 预期行为的哪一条未达成
4. 你的 Claude Code 版本和 skill 安装方式

这能帮助定位问题是 skill 本身的 bug 还是 Claude 版本/配置的差异。
