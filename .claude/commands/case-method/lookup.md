执行 `case-method` skill 的 **lookup** 子命令流程。用户输入：$ARGUMENTS

对应 `case-method/SKILL.md` 中 "### 4. `/case-method:lookup`" 章节的完整流程：

1. **搜索匹配**：读取 `case-method/references/07_术语表与学者争论.md`，对中英文混合查询同时匹配中文术语名、英文术语名和定义关键词
2. **按查询类别补充读取**对应 references 以提供深度上下文：
   - 概念类（拉伸、阶梯、范畴、效度）→ 读取 01
   - 因果类（INUS、反事实、机制、等终性）→ 读取 02
   - 案例选择类（DV 选择、负面案例、可能性原则）→ 读取 03
   - 方法类（过程追踪、一致性、Mill 方法、关键案例）→ 读取 04
3. **同时匹配术语和争论时**：以术语格式为主，在"相关概念"部分简要提及相关争论

**输出格式**：
- **术语**：5 字段（定义 / 核心要点 / 来源 / 相关概念 / 在研究设计中的含义）
- **争论**：4 字段（立场 A / 立场 B / 核心张力 / 对研究设计的含义）

若术语不在知识库中：基于已有知识给简短定义，并明确说明不在本库 17 章核心文献范围内。
