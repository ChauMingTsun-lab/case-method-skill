执行 `case-method` skill 的 **write** 子命令流程。用户输入：$ARGUMENTS

对应 `case-method/SKILL.md` 中 "### 3. `/case-method:write`" 章节的完整流程：

1. **前置信息收集**
   - 若用户已完成 design / check 流程：直接复用其输出信息定制生成
   - 若仅指定主题：先询问研究问题/案例的简要描述 + 具体方法论选择
   - 若主题不明确：展示 8 个支持主题让用户指定

2. **按主题读取对应 references 文件并生成**（参考 SKILL.md 的主题对照表）
   - 案例选择 → 03；过程追踪 → 04；一致性方法 → 04
   - 比较设计 → 03+04；类型学 → 05；概念操作化 → 01
   - 因果推断 → 02；研究设计总论 → 00

3. **输出三段式**：
   - 方法论段落草稿（300-500 字，基于用户具体研究定制）
   - 关键引用列表（作者、年份、核心论点）
   - 审稿人常见质疑与预防性回应（2-3 个典型）

**写作原则**：学术规范语言；引用具体作者年份；承认局限并说明对策；区分方法论辩护与方法论描述。
