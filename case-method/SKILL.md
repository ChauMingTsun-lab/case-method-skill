## 子命令

### 1. `/case-method:design [研究问题描述]`

交互式引导用户完成案例研究设计的关键决策。

**流程**：下方已内联核心决策表，可直接使用。仅在需要 13 步完整检查清单或详细错误说明时，按需读取 `references/06_实操指南.md`。按阶段交互推进。

#### 阶段 A：研究谜题诊断

1. 让用户描述研究问题和因变量
2. 确认目标：causes-of-effects（解释特定案例）还是 effects-of-causes（估计一般效应）？
3. 追问对照空间（contrast space）：为什么这个结果需要解释？与什么比较使它成为谜题？

#### 阶段 B：概念审查

→ 按需读取 `references/01_概念构建与测量.md`

4. 核心概念是否清晰界定？检查 Sartori 阶梯位置（HL/ML/LL）
5. 跨案例应用时是否存在概念拉伸风险？
6. 识别范畴类型 → 推荐相应扩展策略：

| 范畴类型 | 扩展策略                 | 防拉伸方法             |
| -------- | ------------------------ | ---------------------- |
| 经典范畴 | 上移阶梯（减少内涵）     | 保留最小定义特征       |
| 家族相似 | 识别交叉模式             | 不要求普遍共享属性     |
| 辐射范畴 | 添加修饰词（下移到子类） | 子类可只取中心属性子集 |

#### 阶段 C：案例选择

→ 按需读取 `references/03_案例选择与选择偏差.md`

7. 界定范围条件，排除因果异质性区域
8. 按研究目标推荐设计（参考下表）
9. 应用可能性原则评估负面案例（纳入规则 + 排除规则）
10. 评估因果同质性 vs. 推广性权衡

**案例选择快速决策表**：

| 研究目标     | 推荐设计        | DV选择     | 关键注意                          |
| ------------ | --------------- | ---------- | --------------------------------- |
| 假设生成     | 极端/异常案例   | ✓         | 发现的模式需另行检验              |
| 必要条件检验 | 全部正面案例    | ✓         | 单个反例即可推翻                  |
| 充分条件检验 | DV变异案例      | ✗         | 需正面+相关负面案例               |
| 因果效应估计 | 基于IV选择      | ✗         | 需满足条件独立性                  |
| 理论发展     | 最相似+偏差案例 | 部分✓     | 结合案例内分析                    |
| 理论检验     | 关键案例        | 视类型     | most-likely反驳；least-likely支持 |
| 偏差案例分析 | 偏离预期的案例  | N/A        | 精炼理论、发现遗漏变量            |
| 类型学分析   | 基于类型学空间  | 由类型决定 | 聚焦信息量丰富的类型              |

#### 阶段 D：方法选择

→ 按需读取 `references/04_研究方法_过程追踪与一致性.md`

11. 按条件推荐方法（参考下表）
12. 如涉及多变量交互或等终性 → 读取 `references/05_类型学理论与研究设计.md`

**方法选择快速指南**：

| 条件                       | 推荐方法                       | 理由                         |
| -------------------------- | ------------------------------ | ---------------------------- |
| 理论已充分发展，有明确预测 | 一致性方法 → 过程追踪确认     | 直接检验预测，再验证因果链   |
| 理论不发达，变量不确定     | 启发式案例研究 + 过程追踪      | 先发现新变量和机制           |
| 少量案例（2-5个）          | 最相似/最不同 + 过程追踪       | Mill方法提供比较框架         |
| 中量案例（5-15个）         | QCA 或小N统计                  | 布尔代数识别等终性           |
| 复杂因果过程，多中间步骤   | 过程追踪（分析性解释）         | 追踪因果链每一步             |
| 怀疑等终性存在             | 类型学理论                     | 系统枚举变量组合与路径       |
| 单一案例                   | 一致性（多理论竞争）+ 过程追踪 | 案例内检验多个竞争理论       |
| 强力理论检验               | 关键案例分析                   | most-likely/least-likely案例 |

#### 阶段 E：输出研究设计摘要

13. 生成结构化摘要：

```
## 研究设计摘要
- **研究谜题**：[结果] + [对照空间]
- **核心概念**：[概念定义] + [阶梯位置] + [范畴类型]
- **因果理论**：[自变量] → [机制] → [因变量]
- **等终性考量**：[是否存在替代因果路径？若有，列出已识别的路径]
- **案例选择**：[设计类型] + [案例] + [选择依据]
- **分析方法**：[方法] + [选择理由]
- **潜在威胁**：[初步识别，完整评估请运行 `/case-method:check`]
- **范围条件**：[理论适用域]
```

建议将此摘要复制保存，以便后续使用 `/case-method:check` 或 `/case-method:write` 时直接粘贴。

**交互原则**：

- 用户已提供足够信息时，跳过提问，直接输出分析并推进
- 若用户提供了完整研究设计且只需快速反馈，可跳过交互直接输出摘要并建议运行 check
- 每个阶段提出 1-2 个关键问题，等待用户回答后再推进
- 发现方法论问题时立即指出，引用具体文献（作者+年份）
- 提供 2-3 种方案及其权衡，让用户做选择

### 2. `/case-method:check [研究设计描述]`

对照方法论标准评估研究设计的潜在问题。

**流程**：

1. **信息完整性预检**：评估用户输入是否包含三个核心要素（研究问题/自变量、案例选择、分析方法）
   - 若来自 design 摘要：直接使用，无需重新收集
   - 若三要素齐全：直接进入评估
   - 若信息不完整：列出缺失项，提供选择——(a) 补充信息后完整评估 (b) 基于已有信息部分评估（注明缺失维度无法评估）
2. 下方已内联常见错误速查表。仅在需要完整检查清单或详细错误说明时读取 `references/06_实操指南.md`
3. 按以下六个维度逐项评估：

| 维度     | 检查要点                                                                                                       | 按需读取 |
| -------- | -------------------------------------------------------------------------------------------------------------- | -------- |
| 概念     | 概念拉伸？阶梯位置恰当？范畴类型匹配？                                                                         | 01       |
| 案例选择 | DV选择偏差？无关负面案例？可能性原则？声称检验充分条件时是否有DV变异？声称检验必要条件时是否包含全部正面案例？ | 03       |
| 因果推断 | 虚假相关？因果优先性？因果深度？                                                                               | 02, 04   |
| 方法匹配 | 方法与研究目标匹配？是否需要补充方法？                                                                         | 04       |
| 等终性   | 是否考虑多条因果路径？                                                                                         | 02, 05   |
| 推广边界 | 范围条件明确？推断是否过度？因果异质性？                                                                       | 03       |

**输出格式**（每个维度）：

- 🟢 无问题
- 🟡 需注意（存在风险但可辩护）
- 🔴 需修改（存在严重方法论问题）

每个 🟡/🔴 项附：问题描述 → 文献依据 → 修改建议。

六维度评估后输出 `## 综合评估`：列出最优先需修改的 1-3 个问题，每个问题标注对应 design 流程的阶段（如"此问题对应阶段 C，建议重新审视案例选择"），以及建议的下一步行动（如运行 `/case-method:write [相关主题]` 撰写方法论章节）。

**常见错误速查**（内联，无需读取 references）：

| 错误               | 识别信号                             | 防范                           |
| ------------------ | ------------------------------------ | ------------------------------ |
| 概念拉伸           | 概念适用于任何案例；含义跨案例不一致 | Sartori阶梯；识别范畴类型      |
| DV选择偏差         | 样本中因变量无变异；未检查负面案例   | 纳入负面案例或限制推断范围     |
| 忽视等终性         | 使用求同法时假设唯一因果路径         | 类型学理论；主动考虑多路径     |
| 过程追踪≠更多观察 | 声称"增加了N"而不解释逻辑            | 力量在于步骤间逻辑连接         |
| 一致性止于一致性   | 理论预测与结果一致就停止             | 三种威胁评估 + 过程追踪        |
| 纳入无关负面案例   | 负面案例缺乏任何相关自变量           | 可能性原则的纳入/排除规则      |
| 极端案例推广       | 复杂因果路径只在极端条件下存在       | 明确推断边界；新样本检验       |
| 概念争议当测量问题 | 批评者质疑的是系统化概念本身         | 区分 Level 1 与 Level 2-4 问题 |

**引导性自查问题**：展示给用户，完整 8 题见 `references/06_实操指南.md` 第 6.5 节。核心 3 题速查：

- 有什么证据能让你放弃你的假设？（理论可证伪性）
- 你能说出一个同样一致但机制完全不同的替代解释吗？（排除替代假设）
- 你的结论在什么条件下**不**成立？（范围条件）

### 3. `/case-method:write [主题]`

辅助撰写方法论章节，提供规范学术表述和关键引用。

**前置信息收集**：

- 若用户已完成 design 或 check 流程，直接复用其输出信息定制生成
- 若用户仅指定主题但未提供研究设计背景，先询问 (a) 研究问题/案例的简要描述 (b) 具体的方法论选择（如过程追踪的哪种类型），然后再生成定制化段落
- 若主题不明确，展示下表中 8 个支持主题，请用户指定后再执行

**支持的主题**：

| 主题关键词                     | 读取文件 | 输出内容                                   |
| ------------------------------ | -------- | ------------------------------------------ |
| 案例选择 / case selection      | 03       | 设计类型辩护、选择偏差声明、负面案例说明   |
| 过程追踪 / process tracing     | 04       | 定义、类型选择辩护、证据标准、局限性声明   |
| 一致性方法 / congruence        | 04       | 五步程序、三种威胁评估、与过程追踪的结合   |
| 比较设计 / comparative         | 03, 04   | MSSD/MDSD思路、Mill方法局限性声明          |
| 类型学 / typology              | 05       | 构建逻辑、属性空间缩减、与类型学理论的区分 |
| 概念操作化 / concept           | 01       | 概念定义策略、测量效度框架、跨情境策略     |
| 因果推断 / causation           | 02       | 因果观辩护、INUS说明、等终性处理           |
| 研究设计总论 / design overview | 00       | 案例研究定位、四大优势、学科辩护           |

**输出格式**：

1. **方法论段落草稿**（300-500字，基于用户具体研究设计信息定制生成，用户自行调整措辞和引用格式）
2. **关键引用**（作者、年份、核心论点 — 用户自行转为所需引用格式）
3. **审稿人常见质疑与预防性回应**（2-3 个典型质疑）

**写作原则**：

- 学术规范语言，术语精准但不堆砌
- 引用具体作者和年份，展示方法论意识
- 承认局限、说明对策（anticipate criticism）
- 区分方法论辩护（为什么选这个方法）和方法论描述（怎么做的）

### 4. `/case-method:lookup [术语或争论]`

快速查询术语定义或学者争论。

**流程**：

1. 读取 `references/07_术语表与学者争论.md`，搜索匹配内容（对中英文混合查询，同时匹配中文术语名、英文术语名和术语定义中的关键词）
2. 按查询类别补充读取对应 references 以提供更深入的上下文：
   - 概念构建类（概念拉伸、阶梯、范畴类型、测量效度等）→ 同时读取 `references/01`
   - 因果理论类（因果、INUS、反事实、机制、等终性等）→ 同时读取 `references/02`
   - 案例选择类（DV选择、负面案例、选择偏差、可能性原则等）→ 同时读取 `references/03`
   - 分析方法类（过程追踪、一致性方法、Mill方法、关键案例、QCA 等）→ 同时读取 `references/04`
3. 当查询同时匹配术语和争论时，以术语格式为主输出，在"相关概念"部分简要提及相关争论
4. 若术语不在知识库中，基于已有知识给出简短定义，并说明不在本知识库文献范围内

**术语输出格式**：

```
## [中文术语] (English Term)
**定义**：...
**核心要点**：...
**来源**：Author (Year)
**相关概念**：...
**在研究设计中的含义**：...
```

**争论输出格式**：

```
## [争论主题]
**立场A**：[作者] — [核心论点]
**立场B**：[作者] — [核心论点]
**核心张力**：...
**对你的研究设计的含义**：...
```

**六大核心争论索引**（无需读取 references 即可列出）：

1. **推断逻辑统一性**：KKV（统一推断逻辑）vs. G&B（共享认识论、不同方法论）
2. **因果机制 vs. 因果效应**：KKV（因果效应为首）vs. G&B（机制同等重要）
3. **DV选择合法性**：Geddes/KKV（原则上禁止）vs. Collier & Mahoney/G&B（特定目的下合法）
4. **概念修改策略**：Sartori（仅经典阶梯）vs. Collier & Mahon（家族相似、辐射范畴）
5. **负面案例选择**：全案例设计 vs. Mahoney & Goertz（可能性原则）
6. **质性 vs. 量化方法地位**：KKV（量化提供范式）vs. Mahoney & Goertz（各有合法性）

## 无子命令时的行为

当用户直接提出方法论问题（不使用子命令）时：

1. 判断问题类型，参考对应知识库文件（同时匹配多条规则时，合并读取所有相关文件）
2. 若问题需要用户的研究设计信息才能回答（如方法选择、案例选择），先提出 2-3 个关键问题（案例数量、理论成熟度、研究目标），然后基于回答给出建议
3. 简短直接回答，引用具体文献
4. 如问题涉及研究设计决策，建议使用 `/case-method:design` 获得完整引导
5. 若 references 文件无法读取，基于 SKILL.md 内联内容和自身知识回答，并告知用户

**问题类型路由**：

| 用户问题信号                               | 对应文件          |
| ------------------------------------------ | ----------------- |
| "概念定义""概念拉伸""测量效度"             | 01                |
| "因果""INUS""等终性""机制"                 | 02                |
| "选案例""选择偏差""负面案例""可能性原则"   | 03                |
| "过程追踪""一致性方法""Mill方法""关键案例" | 04                |
| "类型学""属性空间""多变量组合"             | 05                |
| "研究流程""检查清单""常见错误"             | 06                |
| "XX是什么意思""XX和YY的争论"               | 07                |
| "比较两种设计""哪种方案更好"               | 06 + 相关主题文件 |

## 通用原则

- **语言**：简体中文，关键术语附英文原文
- **引用**：始终注明作者和年份，不编造引用
- **对立观点**：存在学者分歧时，呈现多方立场而非只取一方
- **实用导向**：每个建议附可操作的下一步
- **深度控制**：基础问题用内联表格直接回答；复杂问题读取 references 提供详细分析
