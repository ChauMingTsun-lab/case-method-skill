# 类型学理论与研究设计

> 本文件是案例研究方法知识库的第五部分，聚焦于类型学理论（typological theory）——一种整合跨案例比较与案例内分析的高级研究框架。类型学理论将变量组合系统化为"类型"，为每种类型指定因果机制和预期结果，从而在一个统一框架内处理等终性、交互效应和复杂因果关系。

## 5.1 类型学 vs. 类型学理论

### 关键区分

George & Bennett (Ch.11) 强调了一个经常被混淆的区分：

**类型学（Typology）**
- 将现象按共享特征划分为不同类型
- 可以是纯描述性的、无理论的
- 只是一种分类工具
- 例：将政治体制分为"民主"、"威权"、"极权"三类——这是分类，但没有解释为什么某些国家属于某个类型

**类型学理论（Typological Theory）**
- 不仅分类，而且通过因果机制和路径将类型与结果连接
- 解释**为什么**特定类型产生特定结果
- 包含条件性概括——"在这种变量组合下，通过这种机制，产生这种结果"

George & Bennett 的正式定义：

> 类型学理论是一种指定**自变量**、划定它们的**类别**、描述每个变量的**个体效应**以及每种**变量组合（配置/类型）**如何产生特定结果的理论。它提供的是**条件性概括**（contingent generalizations）。

### 为什么区分重要

混淆类型学与类型学理论是案例研究中一个常见的方法论错误。如果你只是将案例归类而没有为每种类型指定因果路径，你做的是分类描述而非因果分析。

**检验标准**：你的类型学是否回答了"为什么？"如果它只回答"是什么类型？"，那它还只是类型学，不是类型学理论。

## 5.2 属性空间的构建

### 属性空间的定义

**属性空间**（Property Space）是所有变量可能组合的完整集合——即类型学理论的"全地图"。

如果你有 3 个二元变量（每个有高/低两个值），属性空间就是 2³ = 8 种可能的组合。每种组合代表一个理论上可能的"类型"。

### 归纳式构建

**从案例到理论——建筑砌块法**：

1. **起步**：通过探索性案例研究识别关键变量
2. **方法**：分析归纳法（analytic induction）——虽然理论驱动，但保持对案例中涌现（emerging）的新模式的开放性
3. **积累**：每个案例识别一条不同的因果路径 → 逐步累积成因果路径的库（repertoire）
4. **迭代**：新案例可能修正对已有路径的理解，或揭示全新的路径

**优势**：
- 扎根于经验现实——不会产生与实际案例脱节的"理论幽灵"
- 适合理论不发达的领域

**风险**：
- 类型可能无限增殖——每个新案例似乎都需要一个新类型
- 需要研究者判断何时停止精细化（when is enough enough?）
- 依赖于初始案例的选择——如果初始案例有偏差，整个类型学也会有偏差

### 演绎式构建

**从理论到案例——自上而下法**：

1. **明确研究目标**：你想解释什么？
2. **识别相关理论和变量**：从文献中识别关键解释因素
3. **构建完整属性空间**：列出所有数学上可能的变量组合
4. **理论化**：为每种类型指定预期的因果机制和结果
5. **缩减**（见下节）：将不可能或不相关的类型排除

**优势**：
- 系统性强——不容易遗漏重要的类型
- 显式地处理了等终性——不同类型可以通过不同路径达到相同或不同的结果

**风险**：
- 可能产生理论上存在但经验中不存在的"空单元"（empty cells）
- 但 George & Bennett 指出：空单元本身就有信息价值——为什么这种组合在现实中不存在？分析空单元时应区分三种情况：
  1. **逻辑不可能**：变量组合在逻辑上自相矛盾（如"强大独立工会 + 极端劳工压制"）
  2. **社会结构不可能**：虽然逻辑上可能，但社会机制使其极不可能出现（如"高度工业化 + 完全缺乏城市化"）
  3. **经验上尚未出现**：逻辑和社会结构上都可能，只是恰好没有观察到——这类空单元最有潜在理论价值，可能预示未来出现的新类型

## 5.3 属性空间的缩减

完整的属性空间通常包含太多类型——5 个二元变量就有 32 种组合。研究者必须缩减属性空间以聚焦于最有价值的类型。

### 三种缩减标准

**标准一：移除社会不可能类型**

某些变量组合在逻辑上或社会结构上不可能同时存在——不同变量的取值"过度确定"了不同方向的结果。

例：一个国家同时具有"强大的独立工会"和"极端的劳工压制"——这在逻辑上是矛盾的（如果工会很强大，就不太可能存在极端压制，反之亦然）。这种类型可以被排除为社会不可能的。

**标准二：识别信息量丰富的类型**

不是所有类型对理论检验都同样有价值。最有价值的是：

- **最可能案例**（Most-Likely Cases）：所有变量都指向理论预测的结果 → 如果理论在这里失败，就是强有力的反驳
- **最不可能案例**（Least-Likely Cases）：所有变量都不利于理论预测的结果 → 如果理论在这里仍然成立，就是强有力的支持
- **关键案例**（Crucial Cases）：结合了最可能和最不可能的逻辑

**标准三：匹配研究目标与案例类型**

根据可用案例和研究问题的匹配度来选择类型：
- 如果你的研究目标是检验一个特定理论 → 聚焦于最能区分该理论与竞争理论的类型
- 如果你的研究目标是发展新理论 → 聚焦于空单元和偏差案例
- 如果你的研究目标是政策建议 → 聚焦于与政策情境最相关的类型

## 5.4 四种研究设计

从类型学空间中可以衍生出四种系统性的研究设计：

### 设计一：同类型比较（Same Type, Validating vs. Deviant）

**情形**：两个属于同一类型的案例（变量组合相同）但结果不同

**意义**：
- 如果两个案例在所有已知变量上相同但结果不同 → 一定存在**测量误差**或**遗漏变量**
- 这是发现新变量的强大驱动力
- 研究者需要深入检查：是某个变量被错误编码了？还是有一个未被纳入的关键因素？

**例**：如果类型学预测"强国家 + 弱社会 = 权威发展"，但两个都是"强国家 + 弱社会"的案例中一个实现了快速发展而另一个没有 → 需要寻找解释差异的新变量。

### 设计二：最相似案例比较（Most Similar, Adjacent Types）

**情形**：两个仅在**一个变量**上不同且结果也不同的案例

**逻辑**：
- 这是 Mill 求异法的类型学版本
- 与标准的最相似设计不同的是，类型学理论为每种类型都指定了预期机制 → 可以用过程追踪来验证
- **将过程追踪整合到最相似设计中**大大增强了推断力度

**例**：两个国家在四个变量上相同，但在"精英分裂"这个变量上不同——一个精英分裂、发生了革命；另一个精英团结、没有革命。类型学理论预测了精英分裂通过什么机制导致革命，过程追踪验证这一机制是否实际运作。

### 设计三：最可能 / 最不可能 / 关键案例

**情形**：单个案例，其变量组合使其成为理论最有利或最不利的检验场景

**最可能案例**（Most-Likely）：
- 所有自变量都取了理论预测应该产生 Y 的值
- 如果 Y 没有出现 → 理论面临严重挑战
- 类比：最好的射手在最有利的条件下也打不中靶 → 需要重新审视枪法理论

**最不可能案例**（Least-Likely）：
- 所有自变量都取了理论预测不应该产生 Y 的值
- 如果 Y 仍然出现 → 理论获得极强的支持
- 类比：最差的射手在最不利的条件下都打中了靶 → 理论可能比想象的更健壮

### 设计四：最不同案例比较（Least Similar Cases）

**情形**：两个在大多数变量上不同但**结果相似**的案例

**逻辑**：
- 如果在几乎所有方面都不同的案例中观察到相似的结果 → 它们**唯一共同的变量**可能是因果变量
- 这是 Mill 求同法的类型学版本
- 需要辅以过程追踪来验证识别出的共同变量是否确实通过预测的机制产生了结果

## 5.5 与过程追踪的整合

### 为什么整合是必要的

类型学比较面临与 Mill 方法相同的推断问题：
- 跨案例比较可以识别**候选因果变量**（哪个变量的差异与结果的差异对应？）
- 但它**不能确认**这个变量是否**真正通过预测的机制**产生了结果
- 一致性可能是巧合的

**过程追踪提供的补充**：
- 检验类型学识别的候选因果变量是否在案例内实际运作
- 追踪因果机制的每一步 → 如果机制的每一步都被观察到，因果推断大大增强

### 案例内分析的关键角色

在类型学与过程追踪的整合中，案例内分析发挥四种关键功能：

1. **修正初步分类**：初始分类可能有误——过程追踪可能揭示案例实际上属于不同类型
2. **细化切分点**：变量的高/低切分可能不当——案例内分析可以提供更精确的切分判断
3. **识别遗漏变量**：如果类型学预测与观察不符，案例内分析可以发现被遗漏的关键变量
4. **验证因果机制**：确认理论预测的机制是否在案例中实际运作

### 迭代过程

类型学理论的构建和检验不是一次性的，而是一个**迭代过程**：

```
构建类型学 → 选择案例 → 案例内分析 → 修正类型学 → 选择新案例 → ...
```

在类型学与经验数据之间的迭代降低了任一方法单独使用时的推断错误风险——类型学提供系统性的框架，过程追踪提供因果验证。

## 5.6 应用示例

### Bennett, Lepgold, Unger 的负担分担研究

**背景**：研究 1990-91 年海湾战争中盟国的负担分担行为——为什么有些国家贡献多，有些贡献少？

**类型学构建**：
- 五个解释变量：集体行动逻辑、威胁平衡、联盟依赖、国内政治、官僚政治
- 完整属性空间：2⁵ = 32 种类型（已从 2⁸ = 256 种可能大幅缩减）
- 通过移除社会不可能类型和聚焦信息量丰富的类型进一步缩减

**案例选择**：
- 六个主要贡献国：美国、英国、法国、德国、日本、埃及
- 每个国家代表类型学中的不同类型

**偏差案例的发现**：
- 叙利亚和法国的行为偏离了类型学预测
- 深入分析（过程追踪）发现了一个遗漏变量："**分享战利品**"搭便车动机——一些国家参与不是为了分担负担，而是为了分享胜利的政治和经济收益
- 第二轮研究增加了"学习理论"变量

**方法论启示**：
- 类型学理论引导了案例选择——研究者知道他们在寻找什么
- 偏差案例推动了理论精炼——遗漏变量的发现不是失败，而是贡献
- 迭代过程（第一轮 → 发现偏差 → 修正理论 → 第二轮）是类型学研究的典型模式

### Lange, Mahoney & vom Hau 的殖民主义与发展

**背景**：研究不同类型的殖民统治如何影响后殖民时期的发展轨迹

**类型学框架**：2×2 矩阵

| | 高殖民程度 | 低殖民程度 |
|---|---------|---------|
| **重商主义殖民者（西班牙）** | 发展逆转（从先进到落后） | 相对稳定（维持低发展水平） |
| **自由主义殖民者（英国）** | 发展逆转（从先进到落后） | 相对发展（利用殖民遗产） |

**等终性的处理**：
- **相同结果**（发展逆转）通过**相反机制**产生：
  - 重商主义殖民者：通过贸易垄断和经济榨取逆转了先进殖民地的发展
  - 自由主义殖民者：通过制度移植和族裔分层逆转了先进殖民地的发展
- 这是等终性的典型案例——同一结果，不同机制

**方法的综合运用**：
- **概念形成**："殖民程度"作为质性测量，通过 Adcock & Collier 的效度框架进行辩护
- **案例选择**：39 个英国殖民地 + 18 个西班牙殖民地；范围条件排除非殖民国家
- **一致性方法**：殖民程度是否在预测方向上与发展结果一致？
- **过程追踪**：追踪具体机制——贸易限制、法治建设、族裔分层如何从殖民统治传导到发展结果
- **比较方法**：最不同设计（西班牙 vs. 英国）+ 类型内比较

**偏差案例**：
- 香港是一个偏差案例——高殖民程度但高度发展
- 分析表明香港的特殊条件（小规模、贸易中心地位、独特的历史时机）使其偏离了一般模式
- 这个偏差**确认**了理论的逻辑：当类型学理论的因果条件不满足时，预测的结果也不出现

## 要点总结

1. **类型学理论 ≠ 类型学**：分类不是解释。每种类型必须有对应的因果机制和条件性概括。

2. **属性空间提供系统性**：完整枚举所有可能的变量组合确保不遗漏重要类型——空单元本身就有信息价值。

3. **缩减是必要的但需有原则**：移除不可能类型、聚焦信息量丰富的类型、匹配研究目标——而非随意选择。

4. **四种研究设计各有用途**：同类型比较发现遗漏变量，最相似比较识别因果变量，关键案例提供强检验，最不同比较提炼核心因素。

5. **与过程追踪的整合是核心**：类型学比较识别候选因果变量，过程追踪验证机制——两者互补而非替代。

6. **偏差案例是理论精炼的引擎**：偏离预测的案例不是失败，而是发现遗漏变量和修正理论的机会。
